## 1. 概述

本文档定义主机（G020 充电宝控制板）与从机（TFT 显示屏 MCU）之间的 I2C 通信协议。双方基于此规范进行开发联调。

**主机**: G020 (Master) — 充电宝主控
**从机**: TFT MCU (Slave) — 屏幕驱动板

---

## 2. I2C 物理层参数

| 参数 | 值 | 说明 |
|------|-----|------|
| I2C 角色 | 主机 Master | G020 |
| 从机地址 (7-bit) | **0x2D** | 见 §6.1 |
| 写地址 (8-bit) | (7-bit << 1) \| 0 | 即 0x5A |
| 读地址 (8-bit) | (7-bit << 1) \| 1 | 即 0x5B |
| 通信速率 | 100 kHz (标准模式) | 可选 400kHz |
| SDA 引脚 | **GPIO 待定** | *等待硬件确认* |
| SCL 引脚 | **GPIO 待定** | *等待硬件确认* |

## 3. 通信时序

### 3.1 写操作 (主机 → 从机)

主机向从机指定寄存器写入数据:

```
主机:  START + SLAVE_W + REG_ADDR + DATA[0] + DATA[1] + ... + DATA[N-1] + STOP
从机:              ACK        ACK       ACK               ACK
```

**写单字节示例:**
```
START + [0x5A] + [0x15] + [0x01] + STOP
       └─W──┘   └─reg──┘ └─data─┘
```

**写多字节示例:**
```
START + [0x5A] + [0x20] + [0x12] + [0x34] + STOP
       └─W──┘   └─reg──┘ └─data0┘ └─data1┘
```

### 3.2 读操作 (主机 ← 从机)

主机先发送寄存器地址(伪写)，然后重复起始信号后读取数据:

```
主机:  START + SLAVE_W + REG_ADDR + Repeated-START + SLAVE_R +         + ACK + ... + NACK + STOP
从机:              ACK        ACK                   ACK        DATA[0]  └─送数─┘     DATA[N-1]
```

**读单字节示例:**
```
START + [0x5A] + [0x10] + Repeated-START + [0x5B] + [数据] + NACK + STOP
       └─W──┘   └─reg──┘                    └─R──┘
```

**读多字节示例:**
```
START + [0x5A] + [0x10] + Repeated-START + [0x5B] + [数据0] + ACK + [数据1] + NACK + STOP
       └─W──┘   └─reg──┘                    └─R──┘
```

> 从机必须支持 Repeated-START 条件，收到寄存器地址后内部切换到对应的寄存器指针，然后通过 Repeated-START 切换到读方向开始送数。

## 4. 寄存器地址映射表

寄存器地址采用 **8-bit**，范围 `0x00` ~ `0xFF`，分为以下区域:

| 地址范围     | 用途            | 方向 | 说明                               |
|--------------|-----------------|------|------------------------------------|
| 0x00~0x0F   | 主机信息        | W    | 主机写入固件版本、在线状态          |
| 0x10~0x1F   | 电池数据        | W    | 主机写入SOC、SOH、循环次数等        |
| 0x20~0x3F   | NTC数据         | W    | 主机写入NTC阻值与保护状态           |
| 0x40~0x4F   | 端口数据        | W    | 主机写入C1/C2/USB-A端口状态         |
| 0x50~0x6F   | 电芯/AFE数据    | R    | 主机读取电芯电压、保护状态          |
| 0x70        | 按键事件        | R/W  | 主机读取TFT按键事件                 |
| 0x71~0x7F   | BMS备用寄存器   | R    | BMS code放在TFT侧时使用             |
| 0x80~0x8F   | 系统信息        | R    | 固件版本、在线状态、升级模式        |


### 4.1 主机信息 (0x00~0x0F) — 主机 → 从机

| 地址      | 名称        | 数据长度 | 类型/单位        | 描述                              | 读写 |
|-----------|-------------|----------|------------------|-----------------------------------|------|
| 0x00      | HOST_FW_VER | 2 Byte   | uint16 LE (BCD)  | 主机固件版本 (V3.0 → 0x0300)     | W    |
| 0x02      | ONLine_CRC  | 1 Byte   | 固定0x55         | 用于从机判断主机是否ready          | W    |
| 0x03~0x0F | 预留        | -        | -                | 后续扩展用                        | -    |


### 4.2 电池数据 (0x10~0x1F) — 主机 → 从机

| 地址      | 名称                 | 数据长度 | 类型/单位        | 描述                     | 读写 |
|-----------|----------------------|----------|------------------|--------------------------|------|
| 0x10      | SOC                  | 1 Byte   | uint8, % (0~100) | 电池电量百分比            | W    |
| 0x11      | SOH                  | 1 Byte   | uint8, % (0~100) | 电池健康度                | W    |
| 0x12      | CYCLE_COUNT          | 2 Byte   | uint16           | 循环次数                  | W    |
| 0x14      | charge_remain_time   | 4 Byte   | uint32           | 剩余充满时间(秒)          | W    |
| 0x18      | discharge_remain_time| 4 Byte   | uint32           | 剩余放空时间(秒)          | W    |
| 0x1c      | Res_vbat             | 2 Byte   | uint16           | 采样电阻Vbat（mV）        | W    |
|  ~0x1F | 预留                 | -        | -                | -                        | -    |


### 4.3 NTC数据 (0x20~0x3F) — 主机 → 从机

| 地址      | 名称        | 数据长度 | 类型/单位  | 描述                                              | 读写 |
|-----------|-------------|----------|------------|---------------------------------------------------|------|
| 0x20      | NTC状态     | 1 Byte   | uint8      | BIT[1:0] 00正常 01低温保护 02高温保护（详见下表）  | W    |
| 0x21~0x24 | BAT_NTC1值  | 4 Byte   | uint32, Ω  | 电池NTC阻值，电源主板NTC     → 0x20 [BIT1:0]      | W    |
| 0x25~0x28 | BAT_NTC2值  | 4 Byte   | uint32, Ω  | 电池NTC阻值，BMS主板NTC      → 0x20 [BIT3:2]      | W    |
| 0x29~0x2C | PCB_NTC1值  | 4 Byte   | uint32, Ω  | 板端NTC阻值，电源主板PCB NTC → 0x20 [BIT5:4]      | W    |
| 0x2D~0x30 | PCB_NTC2值  | 4 Byte   | uint32, Ω  | 板端NTC阻值，电源主板PCB NTC → 0x20 [BIT7:6]      | W    |
| 0x31~0x3F | 预留        | -        | -          | -                                                 | -    |

**NTC状态 (0x20) — BIT 定义:**

| BIT域      | 对应的NTC寄存器       | 值域                              |
|------------|----------------------|-----------------------------------|
| [BIT1:0]   | BAT_NTC1 (0x21~0x24) | 00=正常, 01=低温保护, 02=高温保护  |
| [BIT3:2]   | BAT_NTC2 (0x25~0x28) | 00=正常, 01=低温保护, 02=高温保护  |
| [BIT5:4]   | PCB_NTC1 (0x29~0x2C) | 00=正常, 01=低温保护, 02=高温保护  |
| [BIT7:6]   | PCB_NTC2 (0x2D~0x30) | 00=正常, 01=低温保护, 02=高温保护  |


### 4.4 端口数据 (0x40~0x4F) — 主机 → 从机

| 地址 | 名称              | 数据长度 | 类型/单位      | 描述                    | 读写 |
|------|-------------------|----------|----------------|-------------------------|------|
| 0x40 | C1_STATUS         | 1 Byte   | 枚举 (见下方)  | C1 端口状态             | W    |
| 0x41 | C1_VOLTAGE        | 2 Byte   | uint16 LE, mV  | C1 端口电压             | W    |
| 0x43 | C1_CURRENT        | 2 Byte   | uint16 LE, mA  | C1 端口电流             | W    |
| 0x45 | C2_STATUS         | 1 Byte   | 枚举 (见下方)  | C2 端口状态             | W    |
| 0x46 | C2_VOLTAGE        | 2 Byte   | uint16 LE, mV  | C2 端口电压             | W    |
| 0x48 | C2_CURRENT        | 2 Byte   | uint16 LE, mA  | C2 端口电流             | W    |
| 0x4A | USBA_STATUS       | 1 Byte   | 枚举 (见下方)  | USB-A 端口状态          | W    |
| 0x4B | USBA_VOLTAGE      | 2 Byte   | uint16 LE, mV  | USB-A 端口电压          | W    |
| 0x4D | USBA_CURRENT      | 2 Byte   | uint16 LE, mA  | USB-A 端口电流          | W    |
| 0x4F | SMALL_current_mode| 1 Byte   | uint8          | 小电流模式 1打开 0关闭  | W    |

**端口状态枚举 (C1_STATUS / C2_STATUS / USBA_STATUS):**

| 值    | 说明                          |
|-------|------------------------------------|
| 0x00  | 未连接/空闲 (disconnected/standby)  |
| 0x01  | 充电中 (charging)                   |
| 0x02  | 放电中 (discharging)                |
| 0x03~0xFF | 保留 (reserved)                 |


### 4.5 电芯/AFE数据 (0x50~0x6F) — 主机 ← 从机

| 地址      | 名称          | 数据长度 | 类型/单位        | 描述                                   | 读写 |
|-----------|---------------|----------|------------------|----------------------------------------|------|
| 0x50      | V1            | 2 Byte   | uint16 LE, mV    | 电芯1 电压                             | R    |
| 0x52      | V2            | 2 Byte   | uint16 LE, mV    | 电芯2 电压                             | R    |
| 0x54      | V3            | 2 Byte   | uint16 LE, mV    | 电芯3 电压                             | R    |
| 0x56      | V4            | 2 Byte   | uint16 LE, mV    | 电芯4 电压                             | R    |
| 0x58      | VPACK         | 2 Byte   | uint16 LE, mV    | 电池组总电压                           | R    |
| 0x5A      | BAT_CURRENT   | 2 Byte   | int16 LE, mA     | 电池电流(正=充电, 负=放电)             | R    |
| 0x5C~0x5F | NTC1          | 4 Byte   | uint32 LE, Ω     | 温度 NTC1 阻值                         | R    |
| 0x60      | OVP_PERMANENT | 1 Byte   | 0x5A \| bit0     | 系统过压永久失效(需写Flash)            | R    |
| 0x61      | AFE_PROTECT1  | 1 Byte   | Bit 标志         | AFE 状态 STATUS0 (参照规格书 0x00)     | R    |
| 0x62      | AFE_PROTECT2  | 1 Byte   | Bit 标志         | AFE 状态 STATUS1 (参照规格书 0x01)     | R    |
| 0x63      | AFE_PROTECT3  | 1 Byte   | Bit 标志         | AFE 状态 STATUS2 (参照规格书 0x02)     | R    |
| 0x64~0x6F | 预留          | -        | -                | -                                      | -    |

> **OVP_PERMANENT (0x60):** 0x5A 为密钥，bit0 为 1 表示系统过压永久失效，需要写入 Flash。


### 4.6 按键事件 (0x70) — 主机 ← 从机 (R) / 主机 → 从机 (W)

| 地址 | 名称      | 数据长度 | 类型/单位     | 描述     | 读写 |
|------|-----------|----------|---------------|----------|------|
| 0x70 | KEY_EVENT | 1 Byte   | 枚举 (见下方) | 按键事件 | R/W  |

**KEY_EVENT (0x70) — 枚举定义:**

| 值    | 说明                              |
|-------|----------------------------------|
| 0x00  | 无事件                            |
| 0x01  | 按键1 单击    bit0                |
| 0x02  | 按键1 双击    bit1                |
| 0x04  | 按键1 连击1   bit2                |
| 0x08  | 按键1 连击2   bit3                |
| 0x10  | 按键1 长按3S  bit4                |
| 0x20  | 按键1 长按xxS bit5                |
| 0xFF  | 其他无效                          |

注：
1. G020主机 10ms~100ms 轮询此状态，读取走非空事件后清零对应 bit。从机需做好 buff 数据缓存，防止丢事件。
2. 系统硬件设计 Key 按键复用了 SC8815A INT 1ms 下降沿信号，唤醒后主机会通过 I2C 屏蔽 INT，但休眠唤醒时刻从机需做好消抖处理，防止误触发。


### 4.7 BMS备用寄存器 (0x71~0x7F) — 主机 ← 从机

> 本节为备用寄存器，当 BMS code 放在 TFT MCU 侧时使用。

| 地址      | 名称                 | 数据长度 | 类型/单位        | 描述                  | 读写 |
|-----------|----------------------|----------|------------------|-----------------------|------|
| 0x71      | SOC                  | 1 Byte   | uint8, % (0~100) | 电池电量百分比         | R    |
| 0x72      | SOH                  | 1 Byte   | uint8, % (0~100) | 电池健康度             | R    |
| 0x73      | CYCLE_COUNT          | 2 Byte   | uint16           | 循环次数               | R    |
| 0x75      | charge_remain_time   | 2 Byte   | uint16           | 剩余充满时间(秒)       | R    |
| 0x77      | discharge_remain_time| 2 Byte   | uint16           | 剩余放空时间(秒)       | R    |
| 0x79~0x7F | 预留                 | -        | -                | -                     | -    |


### 4.8 系统信息 (0x80~0x8F) — 主机 ← 从机

| 地址      | 名称       | 数据长度 | 类型/单位        | 描述                                                  | 读写 |
|-----------|------------|----------|------------------|-------------------------------------------------------|------|
| 0x80      | FW_VERSION | 2 Byte   | uint16 LE (BCD)  | 固件版本号 (V1.2 → 0x0102)                           | R    |
| 0x82      | ONLINE_CRC | 1 Byte   | 0x55 固定值      | 用于主机判断从机是否ready                              | R    |
| 0x83      | UPDATE_CRC | 1 Byte   | 0xAA 固定值      | 用于主机判断从机进入升级模式(默认00, 写0xAA后200ms进boot)| R    |
| 0x84~0x8F | 预留       | -        | -                | -                                                     | -    |

> **ONLINE_CRC (0x82):** 主机建机后读取此寄存器，值为 0x55 表示从机就绪。
> **UPDATE_CRC (0x83):** 主机写入 0xAA 后，从机延迟 200ms 进入 bootloader 模式。

## 5. 多字节数据字节序

所有多字节数据统一使用 **小端序 (Little-Endian)**:

| 数据                      | 高低字节                                                  | 寄存器地址示例                   |
|---------------------------|-----------------------------------------------------------|----------------------------------|
| 电压 4200mV = 0x1068     | REG_ADDR[0]=0x68 (低), REG_ADDR[1]=0x10 (高)             | 0x50(V1)=0x68, 0x51(V1)=0x10    |
| 固件版本 V2.5 = 0x0205   | REG_ADDR[0]=0x05, REG_ADDR[1]=0x02                       | 0x80=0x05, 0x81=0x02           |


## 6. 使用说明

### 从机(TFT MCU)开发者:
1. 实现 I2C Slave 模式，7-bit 地址 = **0x2D**
2. 收到主机 **写请求**: 解析 REG_ADDR，将后续数据存入对应的寄存器缓冲区供显示使用
3. 收到主机 **读请求**: 从当前 REG_ADDR 指向的寄存器取出数据发送给主机
4. 按键事件产生后，写入 KEY_EVENT (0x70) 寄存器，等待主机读取
5. 每读完一个字节寄存器地址自动递增 (**推荐实现**，支持连续读取)


### 主机(G020)开发者:
1. 确认 I2C 从机地址和 GPIO 引脚后填入代码
2. 定时将主机信息、电池数据、NTC数据、端口数据写入 TFT 寄存器 (§4.1~4.4)
3. 定时读取按键事件 (§4.6)
4. 建机后先读 ONLINE_CRC (0x82) 确认从机就绪
5. 定时读取电芯/AFE数据 (§4.5)
6. 异常状态位定义需根据 UI 要求再明确一下。
